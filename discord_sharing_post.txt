Title: glendix — Gleam bindings for React + Mendix Pluggable Widgets

---

Hi all,

I've been working on building Mendix Pluggable Widgets with Gleam and wanted to share what I came up with.

For those unfamiliar, Mendix is an enterprise low-code platform, and Pluggable Widgets are basically custom React components you can drop into it. They're normally written in TypeScript + JSX.

I wanted to try doing it entirely in Gleam instead — no JSX, no TypeScript, just Gleam compiled to JavaScript.

So I made glendix, a Hex package that provides FFI bindings for React and the Mendix runtime API. It covers React element creation, hooks, event handling, props builder, and most of the Mendix types (EditableValue, ActionValue, ListValue, filters, etc). Mendix types are opaque with FFI accessors, and undefined/Option conversion is handled at the FFI boundary.

A widget ends up looking like this:

```gleam
import glendix/mendix
import glendix/react.{type JsProps, type ReactElement}
import glendix/react/html
import glendix/react/prop

pub fn widget(props: JsProps) -> ReactElement {
  let name = mendix.get_string_prop(props, "sampleText")
  html.div(prop.new() |> prop.class("my-widget"), [
    react.text("Hello " <> name),
  ])
}
```

The compiled JS gets picked up by Mendix's Rollup-based build tools and packaged into an .mpk file that you can deploy to Mendix Studio Pro.

There's also a scaffolding CLI (npx create-mendix-widget-gleam MyWidget) if anyone wants to try it out.

Hex: https://hexdocs.pm/glendix/
GitHub: https://github.com/GG-O-BP/glendix
Example project: https://github.com/GG-O-BP/MendixWidgetGleam

Would love to hear any feedback, especially around the API design. This was my first time publishing a Hex package so I'm sure there's room for improvement.
